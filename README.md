# Multi-Agent Kaggle Solver (LLM-based)

## Описание проекта

Проект представляет собой мультиагентную систему для автоматического решения задач ML в формате Kaggle соревнований.

Оснвонйо цикл представляет собой систему агентов, которые:
- анализируют данные,
- строят модель,
- валидируют результат,
- генерируют submission,
- улучшают решение через цикл обратной связи.

Система полностью автоматизирована: пользователь запускает один скрипт, после чего агенты сами выполняют весь пайплайн.

## Архитектура системы

Система построена как мультиагентный pipeline с центральным управляющим компонентом - Supervisor. Он координирует работу всех агентов и управляет итерационным процессом улучшения решения. Каждый агент выполняет строго определенную функцию, что делает систему более устойчивой и понятной. Взаимодействие агентов организовано в виде последовательного конвейера с возможностью обратной связи. Такая архитектура позволяет постепенно улучшать качество модели без ручного вмешательства.

## Feedback Loop

В основе системы лежит цикл обратной связи, который позволяет автоматически улучшать решение. После генерации кода он запускается, проверяется и при необходимости исправляется. Если код не выполняется, подключается Debugger, который пытается исправить ошибки. После успешного выполнения Critic анализирует результат и предлагает улучшения. Затем Engineer использует эти рекомендации в следующей итерации. Такой подход имитирует реальный процесс разработки модели.

## Оценка модели

Для оценки используется фиксированный holdout, который создается один раз и затем используется во всех итерациях. Это позволяет корректно сравнивать модели между собой и избегать случайных колебаний метрики. В качестве метрики используется RMSE или MAE, в зависимости от задачи. Результат выводится в стандартизированном формате, что позволяет автоматически его извлекать. Такой подход делает процесс оценки стабильным и воспроизводимым.

## Работа с submission

Каждая итерация генерирует собственный submission, который сохраняется отдельно. Это позволяет избежать потери хороших результатов при неудачных экспериментах. После каждой итерации происходит сравнение метрики с текущим лучшим результатом. Если новая модель показывает лучший результат, submission обновляется. В противном случае сохраняется предыдущий лучший вариант. Таким образом обеспечивается сохранность оптимального решения.

## Запуск проекта

Для запуска необходимо установить зависимости и указать API ключ. После этого достаточно выполнить основной скрипт, который запустит всю систему. Все этапы - от анализа данных до генерации submission - выполняются автоматически. Пользователю не требуется вмешиваться в процесс. Это делает систему удобной для быстрого прототипирования решений.

## Используемые модели

В проекте используются модели семейства Qwen. Для генерации кода применяется модель qwen3-coder-next, так как она лучше справляется с синтаксисом и структурой программ. Для анализа и планирования используется qwen3-max-thinking, которая показывает более высокое качество рассуждений. Такое разделение ролей между моделями позволяет повысить общую стабильность системы. Каждая модель используется в той задаче, где она наиболее эффективна.

## Open Source

Модели Qwen являются open-weight и доступны для использования в open-source проектах. Они могут запускаться локально или использоваться через API, такие как RouterAI или OpenRouter. Это соответствует требованиям курса, где необходимо использовать открытые модели. Выбор этих моделей обусловлен балансом между качеством и доступностью. Таким образом, проект не зависит от проприетарных решений.

## Надежность

В системе реализованы базовые механизмы повышения устойчивости. Используется фиксированный split данных, единый preprocessing и контроль времени выполнения. Также добавлен Debugger, который автоматически исправляет ошибки выполнения. Дополнительно применяется очистка output модели перед запуском кода. Эти меры значительно снижают количество падений и делают систему стабильнее.

## Benchmarking

Все результаты сохраняются и логируются, что позволяет анализировать прогресс модели. Для каждой итерации фиксируется метрика и сохраняется код. Это дает возможность сравнивать разные подходы и отслеживать улучшения. Лучший результат сохраняется отдельно и не перезаписывается. Такой подход упрощает анализ и делает эксперименты воспроизводимыми.

## Соответствие требованиям

Проект соответствует основным требованиям курса. В нем реализована мультиагентная архитектура, автоматизация пайплайна и feedback loop. Используются языковые модели и встроена система оценки качества. Также обеспечена воспроизводимость и документированность решения. Это делает проект полноценным примером интеллектуальной агентной системы.
